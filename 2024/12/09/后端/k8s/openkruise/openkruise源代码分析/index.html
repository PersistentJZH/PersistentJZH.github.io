Rollout Reconcile

batchRelease

k apply -f kruise.yaml -> Rollout Reconcile -> 通过rollout找到workloadRef -> 给workloadRef添加label:rollouts.kruise.io/workload-type: deployment -> calculateRolloutStatus（set rollout Phase为Healthy） -> 更新rollout状态 -> 返回

开始灰度发布
k apply -f old.yaml -> webhook回调 /mutate-apps-v1-deployment -> 根据deployment找到对应的rollout对象 -> 根据deployment获取对应的ReplicaSet（stableRS和canaryRS）-> 找到stableRS -> 给deployment添加label rollouts.kruise.io/stable-revision：stableRS[pod-template-hash] -> 给deployment添加Paused=true -> 给deployment添加annotation=rollouts.kruise.io/in-progressing: ‘{“rolloutName”:”rollout-blue”}’ -> 返回 -> 变更写入到etcd ->

->同时触发deployment的Reconcile -> 同步真实deployment状态到etcd -> 给deployment添加ExtraStatus= rollouts.kruise.io/deployment-extra-status: ‘{“updatedReadyReplicas”:1,”expectedUpdatedReplicas”:1}’ -> 判断是否DeploymentRolloutSatisfied -> 如果不满足就requeue

->同时触发rollout的Reconcile -> 根据deployment的 annotation=rollouts.kruise.io/in-progressing: ‘{“rolloutName”:”rollout-blue”}’ -> set rollout 的Phase为Progressing -> creates a new rollout condition (condition_type = Progressing, condition_reason = Initializing) -> 下一轮Reconcile -> 根据cond进入reconcileRolloutProgressing流程 -> runCanary
